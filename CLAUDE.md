# CLAUDE.md — APP_CRDL

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 프로젝트 안내서입니다.

## 프로젝트 개요

**CRDL**(차례대로)은 30년 전 모눈종이에 설계했던 격자 퍼즐 게임을 모바일 앱으로 되살리는 프로젝트입니다.
"앱튜버"(앱 개발 + 유튜브) 활동의 첫 출시작이며, 개발 과정 자체를 콘텐츠로 삼는 *Build in Public* 전략을 씁니다.

플레이어는 커서로 격자를 훑으며 숫자를 **1 → N 순서대로** 먹습니다. 지나간 칸은 다시 지날 수 없고,
길이 엉켜 갇히면 게임 오버입니다.

## 게임 규칙 (장르 2종)

두 장르 모두 공통: 숫자 1 칸에서 출발, 상하좌우로만 이동, 지나간 칸 재통과 불가,
숫자는 "지금 차례"일 때만 밟을 수 있음(순서 게이트).

- **땅파기 (fill)**: 격자의 **모든 칸**을 채우면서 숫자를 순서대로 완성해야 클리어. 갇힌 칸이 생기면 게임 오버.
  수학적으로 "숫자 웨이포인트를 순서대로 지나는 해밀턴 경로(Hamiltonian path)" 문제.
- **길잇기 (path)**: 숫자만 순서대로 이으면 클리어. 모든 칸을 채울 필요는 없음(더 쉬움).

> 참고: 땅파기 모드는 LinkedIn의 "Zip" 퍼즐과 규칙이 사실상 동일합니다. 차별점은 드래그 방식이 아니라
> **방향키/스와이프로 전진하고 갇히면 실패하는 액션성**(스네이크에 가까움)입니다. 이 정체성을 유지하세요.

## 기술 스택

- **앱**: Flutter / Dart (크로스플랫폼, 기존 Android 경험 활용)
- **툴링**: Python (레벨 생성·검증 파이프라인). 이 저장소의 `tools/`가 여기에 해당.
- **프로토타입**: `prototype/crdl.html` (단일 파일, 브라우저에서 바로 플레이 가능한 레퍼런스 구현)

작업 언어 기본값은 **Python**입니다. 새 스크립트/도구는 특별한 이유가 없으면 Python으로 작성하세요.

## 데이터 형식 (`data/levels.json`)

```jsonc
{
  "version": 1,
  "count": 200,
  "levels": [
    {
      "id": 1,
      "difficulty": "easy",        // easy | normal | hard | expert
      "rows": 6, "cols": 5,
      "numbers": [                 // 격자에 심는 숫자들
        { "r": 5, "c": 4, "n": 1 },
        { "r": 2, "c": 1, "n": 2 }
        // ...
      ],
      "start": [5, 4],             // 숫자 1의 위치 (출발점)
      "solution": [[5,4],[5,3], ] // 정답 경로(칸 순서). 힌트/재검증용
    }
  ]
}
```

- 좌표는 `(r, c)` = (행, 열), 0-based. `r`은 위→아래, `c`는 왼쪽→오른쪽.
- 하나의 레벨 데이터는 **두 장르에 모두** 사용 가능(땅파기로 풀리면 길잇기로도 풀림).
- `solution`은 땅파기 기준 정답 경로 하나이며, 힌트 기능이나 재검증에 쓰세요. 유일 정답은 아닐 수 있습니다.
- 난이도 분포: easy 50 / normal 60 / hard 50 / expert 40.

## 레벨이 "항상 풀리는" 이유 (핵심 원리)

정답을 먼저 만들고 숫자를 나중에 심습니다.

1. `snake`(지그재그)로 격자 전체를 지나는 해밀턴 경로에서 출발.
2. `backbite` 연산(양 끝을 접어 뒤집기)을 반복해 해밀턴 성질을 유지한 채 무작위화.
3. 그 경로를 따라 숫자 1..N을 순서대로 심음(1은 시작, N은 끝).

=> 생성된 경로 자체가 정답이므로 **반드시 풀리는** 레벨이 보장됩니다. 옛날 패턴을 복원할 필요가 없습니다.

## 자주 쓰는 명령

```bash
# 레벨 생성 (예: 8x6, 숫자 1~10, 200개)
python tools/crdl_levelgen.py --rows 8 --cols 6 --numbers 10 --count 200 --out data/levels.json

# 생성기 자체 검증
python tools/crdl_levelgen.py --self-test

# 특정 패턴이 풀리는지 확인 (옛 패턴 복원/직접 설계 시)
python -c "from tools.crdl_solver import solve, parse_grid; \
R,C,n=parse_grid('1 . . 4\n. . . .\n2 . . 3'); print(bool(solve(R,C,n,'fill')))"

# 프로토타입 플레이: prototype/crdl.html 을 브라우저로 열기
```
